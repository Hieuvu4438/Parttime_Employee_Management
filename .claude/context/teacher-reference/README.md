# Teacher Reference Materials

These files contain teacher-provided sample materials for an **unrelated hotel management system**.

## Important boundary

This reference is **not** the Parttime Employee Management project. It must not be treated as source truth for the current project.

Use it only to learn:

- the expected structure of each software engineering phase;
- the level of detail expected in scenarios, diagrams, and explanations;
- how to transform natural language requirements into use cases, analysis classes, design classes, database design, sequence diagrams, and black-box test cases;
- the wording pattern for academic report sections.

Do not reuse directly:

- hotel domain entities such as Hotel, Room, Client, Booking, Bill, Service;
- hotel actors such as Manager, Seller, Receptionist, Administrator unless the current project independently has equivalent roles;
- hotel database tables or test data;
- hotel UI names such as SearchRoomFrm or ConfirmFrm.

## Files

1. `01_business-model-natural-language.md` — natural language business model format.
2. `02_use-case-uml.md` — general and detailed use case diagram method.
3. `03_analysis-phase.md` — scenarios, entity extraction, analysis class diagram, and analysis sequence diagram.
4. `04_design-phase.md` — design entity classes, database design, design class diagram, and design sequence diagram.
5. `05_testing-phase.md` — black-box test plan and standard test case format.
6. `06_coding-java-swing-dao.md` — Java Swing / DAO coding patterns from the teacher's hotel demo source code.
